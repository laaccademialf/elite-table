# Elite Table - Структура проекту

## Архітектура

Проект був реорганізований з монолітної структури на модульну архітектуру для кращої масштабованості та підтримуваності.

## Структура папок

```
src/
├── components/          # Переиспользуємі UI компоненти
│   ├── CustomCalendar.jsx     # Календар для вибору дат
│   ├── Footer.jsx             # Нижній колонтитул
│   ├── Navbar.jsx             # Навігаційна панель
│   └── SafeImage.jsx          # Компонент для безпечного завантаження зображень
│
├── constants/           # Константи і початкові дані
│   └── initialData.js        # Категорії товарів, приклади товарів
│
├── context/             # React Context для глобального стану
│   ├── AppContextDefinition.jsx  # Визначення контексту
│   ├── AppProvider.jsx           # Провайдер з логікою управління станом
│   └── useAppContext.jsx         # Hook для користування контекстом
│
├── services/            # API сервіси
│   ├── geminiApi.js    # Функції для роботи з Gemini API (текст і TTS)
│   ├── gemini.js       # Базові налаштування Gemini
│   └── firebase.js     # Firebase конфігурація
│
├── utils/               # Утиліти і допоміжні функції
│   ├── audioUtils.js   # Конвертація PCM в WAV
│   └── iconUtils.jsx   # Утиліти для роботи з іконами
│
├── views/               # Основні представлення сторінок
│   ├── AdminPanel.jsx      # Панель адміністратора
│   ├── CartView.jsx        # Перегляд кошика
│   ├── CheckoutView.jsx    # Оформлення замовлення
│   ├── HomeView.jsx        # Головна сторінка
│   ├── ItemDetailView.jsx  # Деталі товару
│   └── PostItemView.jsx    # Форма додавання нового товару
│
├── App.jsx              # Основний компонент додатку
├── main.jsx             # Точка входу додатку
└── index.css            # Глобальні стилі
```

## Розділення відповідальності

### Context (`context/`)
- **AppContextDefinition.jsx**: Чистий контекст (не містить логіки)
- **AppProvider.jsx**: Компонент-провайдер зі всією логікою управління станом (створення замовлень, управління кошиком, итерactions з Gemini API)
- **useAppContext.jsx**: Custom hook для доступу до контексту в компонентах

### Components (`components/`)
Маленькі, переиспользуємі UI компоненти:
- Календар для вибору дат
- Обробка невдалого завантаження зображень
- Навігаційна панель
- Нижній колонтитул

### Views (`views/`)
Основні сторінки додатку:
- **HomeView**: Головна сторінка з товарами і фільтрами
- **CartView**: Перегляд кошика з концепцією поточної
- **ItemDetailView**: Деталі конкретного товару
- **AdminPanel**: Управління товарами і замовленнями
- **CheckoutView**: Оформлення замовлення
- **PostItemView**: Форма для додавання нового товару

### Services (`services/`)
Зовнішні API інтеграції:
- **geminiApi.js**: Функції для роботи з Gemini (генерація текстів, TTS)
- Інші сервіси (firebase, gemini конфіги)

### Utils (`utils/`)
Допоміжні функції:
- **audioUtils.js**: Конвертація аудіо форматів
- **iconUtils.jsx**: Утиліти для роботи з іконами

### Constants (`constants/`)
Статичні дані і конфігурація:
- Категорії товарів
- Приклади товарів
- Форми для нових товарів

## Переваги нової структури

1. **Масштабованість**: Легко додавати нові views, components і services
2. **Переиспользуємість**: Компоненти і утиліти можна використовувати в різних місцях
3. **Тестування**: Кожна частина має чітку відповідальність
4. **Обслуговування**: Легко знайти і оновити код
5. **Читаємість**: Структура відбиває логіку додатку

## Залежності

- React 19.2
- Lucide React (іконки)
- Tailwind CSS (стилізація)
- Vite (збірка)
- Firebase (автентифікація, база даних)

## Gemini API

Для роботи з Gemini API потрібно встановити змінну окремого середовища:

```
VITE_GEMINI_API_KEY=your_api_key
```

Використовується для:
- Генерації описів товарів через AI
- Генерації концепцій подій
- Text-to-Speech для читання концепцій українською мовою
